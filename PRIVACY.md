# Политика конфиденциальности Nemefisto

**Дата вступления в силу:** 2026-05-01

Nemefisto — это VPN-клиент с открытым исходным кодом, разработанный
с принципом *zero-telemetry*. Этот документ объясняет, какие данные
приложение обрабатывает, где они хранятся и кому передаются.

## Краткая версия

Nemefisto **не собирает, не передаёт и не хранит** на сторонних серверах
никаких данных о пользователе или его активности. Всё, что приложение
читает или пишет, остаётся на вашем компьютере. У нас нет аналитики,
телеметрии, отчётов о крашах, A/B-тестов и прочих механизмов сбора
информации.

## Что приложение делает на вашем компьютере

### 1. Локальные файлы и хранилища

| Что | Где | Зачем |
|---|---|---|
| Серверы подписки (кеш) | `%LOCALAPPDATA%\NemefistoVPN\servers.json` | Чтобы при запуске показать список серверов до обновления подписки |
| Routing-профили | `%LOCALAPPDATA%\NemefistoVPN\routing\*.json` | Кеш правил маршрутизации |
| Geofiles (`geoip.dat`/`geosite.dat`) | `%LOCALAPPDATA%\NemefistoVPN\geofiles\` | Базы IP-адресов и доменов для split-routing |
| Резервная копия системного прокси | `%LOCALAPPDATA%\NemefistoVPN\proxy_backup.json` | Чтобы вернуть оригинальные настройки прокси при отключении |
| Логи (`xray-stderr.log`, helper Event Log) | `%TEMP%\NemefistoVPN\` и Windows Event Log | Локальная диагностика |
| Crash-dump (если случилась паника) | `%LOCALAPPDATA%\NemefistoVPN\crashes\<timestamp>.txt` | Чтобы при следующем старте приложение могло предложить вам выгрузить диагностику |
| Настройки UI | `localStorage` встроенного WebView2 | Темы, фоны, флаги пользовательского опыта |
| URL подписки и HWID-override | Windows Credential Manager | Защищённое хранилище ОС |

Никакие из этих файлов не покидают ваш компьютер. Если вы захотите
поделиться диагностикой с поддержкой, в Settings → «системное» есть
кнопка **«выгрузить диагностику»** — она соберёт zip-архив, который
вы можете отправить вручную (или не отправить).

### 2. Сетевые запросы, которые делает приложение

Nemefisto обращается к сети **только** в этих случаях:

- **Получение подписки.** HTTP(S)-запрос к URL подписки, который вы
  вписали сами. Заголовки запроса описаны в [README](README.md);
  передача `x-hwid` отключается тумблером в Settings → «подписка».
- **Скачивание geofiles.** Только при включенном split-routing —
  загружается `geoip.dat`/`geosite.dat` с GitHub Releases
  (`Loyalsoldier/v2ray-rules-dat`). Запрос идёт **в обход VPN**
  (no-proxy), хеши `.sha256` проверяются перед перезаписью.
- **Routing-профили из URL.** Если в подписке указан
  `autorouting`-источник, профиль скачивается оттуда по расписанию.
- **Leak-test (опционально, по умолчанию on).** После подключения
  делается запрос к `cdn-cgi/trace.cloudflare.com` и `ipwho.is` через
  активный туннель — чтобы убедиться что VPN реально работает и
  показать вам внешний IP. Tумблер в Settings → «безопасность».
- **VPN-трафик.** Само собой, ваш сетевой трафик идёт через VPN-сервер
  выбранной подписки. Это её нормальная функция, выбор сервера и
  доверие к нему — на вашей стороне.

Никаких других серверов приложение не запрашивает. Никакой телеметрии,
crash-репортинга «домой», обновлений конфигурации с центрального
сервера, A/B-тестов, рекламы.

### 3. Системные изменения

- **Системный прокси.** В режиме *proxy* приложение записывает наши
  адреса в реестр HKCU (Internet Settings → ProxyServer/ProxyEnable).
  При отключении значения возвращаются как было; при крахе
  приложения — самовосстановление при следующем запуске.
- **WinTUN-адаптер.** В режиме *TUN* через helper-сервис создаётся
  виртуальный адаптер. После disconnect он удаляется.
- **WFP-фильтры (Windows Filtering Platform).** Если включён kill
  switch — helper-сервис ставит фильтры брандмауэра на уровне ядра.
  Они автоматически снимаются при отключении (DYNAMIC session) и
  cleanup'ятся даже если процесс падает.
- **Helper-сервис (NemefistoHelper).** Устанавливается при первой
  необходимости (TUN или kill switch), работает с правами SYSTEM,
  слушает только локальный named-pipe (`\\.\pipe\nemefisto-helper`)
  с access-check'ами. Удаляется при удалении приложения.

## Что приложение **не делает**

- Не отправляет ID пользователя или устройства на наши серверы
  (потому что у нас нет «наших серверов» в смысле центральной
  инфраструктуры).
- Не отслеживает, какие сайты вы посещаете. Лог Xray работает в
  режиме `none` — содержимое запросов не пишется.
- Не передаёт crash-dump'ы автоматически. Они лежат локально, пока
  вы их сами не удалите или не приложите к bug-report'у.
- Не подключается к интернету, если вы не нажали «подключить» и не
  включили `refreshOnOpen` / `pingOnOpen` в Settings.
- Не имеет remote-control механизмов. Deep-link и заголовки подписки
  имеют **строгий whitelist** — они не могут запускать процессы,
  читать файлы вне стандартных путей, отключать Settings или
  скрывать серверы.

## HWID и идентификация

Если в Settings → «подписка» включен флаг «отправлять HWID» (default
on, потому что Marzban-панели часто его требуют для лицензионного
учёта), при запросе подписки в заголовке `x-hwid` уходит хеш
Windows MachineGuid. Это значение **уникально для вашего устройства**,
но не содержит личной информации (имени пользователя, MAC-адреса
и т.п.) и не передаётся никому, кроме сервера вашей подписки.

Вы можете отключить отправку HWID полностью или вписать произвольное
значение в поле «override» (advanced).

## Заголовки HTTP-запроса подписки

При запросе подписки клиент отправляет:

```
User-Agent: <effective UA>
Accept: */*
Accept-Language: ru-RU
x-app-version: <semver>
x-device-locale: ru-RU
x-client: Nemefisto
```

И опционально (при включенном HWID):

```
x-hwid: <hash>
x-device-os: Windows
x-ver-os: <версия>
x-device-model: <модель>
```

Это всё. Никаких трекеров, рефереров, fingerprint'а браузера и т.п.
не передаётся.

## Третьи стороны

- **Сервер вашей подписки.** Мы не имеем к нему отношения и не несём
  ответственности за его политику конфиденциальности. Уточняйте
  у провайдера подписки.
- **GitHub** (cкачивание geofiles). Стандартные access-логи Cloudflare/
  GitHub. См. их собственные политики.
- **Cloudflare** (для leak-test, опционально). Только при leak-test —
  один HTTP-запрос к `cdn-cgi/trace`.

Ни один из этих сервисов не получает информации, что запрос делает
*именно* Nemefisto или *именно* ваш экземпляр приложения (User-Agent
содержит обычное имя HTTP-клиента, не уникальный ID).
